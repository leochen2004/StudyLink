## 系统功能模块

StudyLink 平台包含三个主要角色：学生、教师和管理员，各端功能如下：

### 学生端

1.  **学生选课**: 浏览并订阅感兴趣的课程，管理已选课程。
2.  **资源下载**: 下载教师发布的课件及学习资料。
3.  **资源搜索**: 通过关键词快速检索所需的学习资源。
4.  **资源查看**: 在线查看资源详情及状态。
5.  **消息通知**: 接收系统通知及教师对提问的回复提醒。
6.  **互动问答**: 在课程下提问，参与讨论，并查看教师的专业回复。
7.  **个人存储**: 上传个人学习资源（私有云存储），支持随时下载。
8.  **个人中心**: 编辑并保存个人资料（头像、简介等）。

### 教师端

1.  **授课管理**: 查看自己负责教学的课程列表。
2.  **资源发布**: 上传教学资源，并可灵活设置资源可见性（公开/私有）。
3.  **资源管理**: 浏览并管理自己发布的所有资源及其状态。
4.  **问答管理**:
    - **待处理问题**: 高效回复学生的课程提问。
    - **历史记录**: 查看并管理历史问答记录，支持撤回或修改。

### 管理员端

1.  **课程管理**: 添加、编辑及维护系统内的所有课程信息。
2.  **教师管理**: 录入新教师账号，管理现有教师列表及信息。
3.  **资源审核**: 对平台上传的所有资源进行合规性审核与监管。
4.  **问答审核**: 监控问答区内容，维护良好的社区讨论环境。

## 项目启动指南

### 1. 环境准备

在启动项目之前，请确保你的本地开发环境满足以下要求：

- **JDK**: 1.8 (必需)
- **Maven**: 3.x (用于构建和依赖管理)
- **MySQL**: 8.0+ (建议 8.0.33)

### 2. 数据库配置

本项目使用 MySQL 数据库。

1.  **初始化数据库**:
    使用项目根目录下的 `schema.sql` 脚本创建数据库和表结构。

    ```bash
    mysql -u root -p -e "CREATE DATABASE IF NOT EXISTS study_link_db;"
    mysql -u root -p study_link_db < study_link_db.sql
    ```

2.  **配置连接信息 (.env.local)**:
    为了安全和便利，建议在项目根目录下创建一个名为 `.env.local` 的文件（该文件已被 git 忽略），并填入你的本地数据库配置：
    ```properties
    # .env.local 示例
    DB_URL=jdbc:mysql://localhost:3306/study_link_db?useSSL=false&serverTimezone=UTC&allowPublicKeyRetrieval=true
    DB_USERNAME=你的用户名 (例如 root)
    DB_PASSWORD=你的密码
    ```
    _如果不创建此文件，系统将默认尝试使用 `root` / `123456` 连接。_

### 3. 启动项目

本项目集成了 Tomcat 7 Maven 插件，支持一键启动。

在项目根目录下运行：

```bash
mvn tomcat7:run
```

启动成功后，访问地址：[http://localhost:8081/studylink](http://localhost:8081/studylink)
